Paper category

Domain & keywords

Pl
Stage 2 — Planning & outline generation
LLM builds a full section scaffold with word-budget targets and argument maps
Brief › LLM planner › Outline + budgets › Human review › Approved outline
Section scaffolding
LLM proposes sections (Abstract · Intro · Related Work · Methods · Results · Discussion · Conclusion) with sub-headings tailored to paper type
Word & figure budgets
Target length per section, estimated table count, image/figure placement markers inserted as [FIG:] placeholders
Argument mapping
High-level thesis → supporting claims → evidence slots; ensures logical coherence before any writing begins
Citation blueprint
Key reference slots marked as [CITE:topic] so the RAG engine knows exactly what evidence to surface per section
Human can edit the outline before generation proceeds — all changes persist downstream.
Db
Stage 3 — Background data selection
Build the knowledge base that feeds RAG retrieval for all generation steps
Upload PDFs › Chunk + embed › Vector index › KG layer › Ready for RAG
Document ingestion
Upload PDFs, DOCXs, arXiv links — auto-chunked and embedded into a vector store (FAISS / Chroma / Pinecone)
Structured datasets
CSV / JSON datasets, citation DBs (Semantic Scholar, Crossref, PubMed) queried and indexed alongside prose
Web crawl seeds
Seed URLs and search queries — crawled content embedded into the retrieval index for broader coverage
Knowledge graph
Entity extraction builds a lightweight KG; authors, papers, and concepts linked for relationship-aware retrieval
Index stored per-project and reused across all section generation calls — no re-embedding needed.
Gn
Stage 4 — Tri-engine section generation
RAG retrieval, LLM world knowledge, and live web search fused per section — with tables and images
RAG retrieval LLM knowledge Live web search Table engine Image / figure gen
Section slot › Retrieve › Draft › Self-critique › Revise › Next section
RAG retrieval
Each [CITE:] slot triggers k-NN search; top passages injected as grounded context into the writer LLM call
LLM world knowledge
Base model contributes background reasoning, methodology descriptions, and synthesis beyond the uploaded docs
Live web search
Real-time queries fill gaps: latest stats, recent papers past the training cutoff, breaking results — cited inline
Table generation
Metrics, comparisons, and ablations auto-formatted into Markdown / LaTeX tables; column headers inferred from data
Image & figure gen
Matplotlib / Seaborn charts from datasets; architecture diagrams as SVG; concept figures via generative model at [FIG:] slots
Section-by-section loop
Each outline section runs its own retrieve → draft → self-critique → revise cycle before appending to manuscript
All three text engines run concurrently; a fusion LLM call merges and deduplicates results before writing.
Ch
Stage 5 — Cross-section coherence check
Automated consistency, citation, and logical-flow validation across the full assembled draft
Full draft › LLM auditor › Flag list › Auto-fix › Author review › Approved
Logical flow audit
LLM reads the full draft end-to-end; flags contradictions between sections and non-sequitur transitions
Citation validation
Every [CITE] resolved against the index; duplicates merged; format normalised to target style; missing refs flagged
Terminology consistency
Acronym list auto-built; first-use expansion enforced; variant spellings unified (e.g. "fine-tuning" vs "fine tuning")
Figure & table refs
Every "Figure X" and "Table Y" cross-checked against actual count; numbering auto-corrected throughout
Abstract ↔ body sync
Abstract claims verified to be supported in body; contributions in Introduction matched against Results section
Human review queue
All unresolved flags shown in a diff-style UI for the author to accept, reject, or rewrite before export
Issues resolved iteratively — a re-check pass runs automatically after each batch of fixes.
Ex
Stage 6 — Document export
Generate production-ready .docx and .pdf with all figures, tables, and citations properly embedded
Approved draft › Template loaded › Assemble › Embed media › .docx + .pdf ZIP
.docx generation
python-docx assembles: title page, ToC, styled headings, body text, inline citations, embedded figures, formatted tables, reference list
.pdf generation
LaTeX pipeline (Pandoc → pdflatex) or WeasyPrint HTML→PDF; supports journal templates (IEEE, ACM, Springer, Elsevier)
Figure embedding
All generated charts and images placed at [FIG:] markers; captions auto-populated; figure list appended at end
Table formatting
Markdown tables → native DOCX tables or LaTeX tabular; zebra-striping applied; column widths auto-fitted to content
Supplementary pack
Appendices, raw data tables, and extended figures bundled as a separate file or merged as appendix pages
Parallel output
Both .docx and .pdf generated simultaneously; delivered as a single ZIP download — no extra conversion step needed
Both formats generated in parallel using the same source AST — no conversion artifacts between them.
