HyDE (hypothetical document embedding)
Before embedding the query, ask the LLM to write a hypothetical answer. Embed that answer instead — it sits much closer to real documents in embedding space than a short keyword query.
Hybrid search (dense + sparse)
Dense k-NN catches semantic matches ("cardiovascular disease" ↔ "heart failure"). BM25 catches exact keyword matches (author names, model IDs, acronyms). Neither alone is sufficient.
Reciprocal rank fusion (RRF)
Merges ranked lists from dense and sparse retrieval without needing normalised scores. Formula: score = Σ 1/(k + rank_i) where k=60. Simple and robust.
Cross-encoder reranking
A second, slower model reads (query, chunk) pairs together and assigns a relevance score. Significantly improves precision — run on top-20 candidates to get the final top-5.
Document ingestion & chunking
# requirements: langchain, pypdf, sentence-transformers, faiss-cpu, rank-bm25 from langchain.document_loaders import PyPDFLoader, UnstructuredURLLoader from langchain.text_splitter import RecursiveCharacterTextSplitter from langchain.embeddings import OpenAIEmbeddings from langchain.vectorstores import FAISS from rank_bm25 import BM25Okapi import pickle, json def build_index(pdf_paths: list[str], urls: list[str]) -> dict: # 1. Load documents docs = [] for path in pdf_paths: docs.extend(PyPDFLoader(path).load()) if urls: docs.extend(UnstructuredURLLoader(urls).load()) # 2. Chunk — 512 tokens, 64-token overlap, respect sentence boundaries splitter = RecursiveCharacterTextSplitter( chunk_size=512, chunk_overlap=64, separators=["\n\n", "\n", ". ", " "] ) chunks = splitter.split_documents(docs) # 3. Attach metadata (source, page, section heading if detectable) for i, c in enumerate(chunks): c.metadata["chunk_id"] = i # 4. Dense embeddings → FAISS vector store embedder = OpenAIEmbeddings(model="text-embedding-3-small") vector_store = FAISS.from_documents(chunks, embedder) vector_store.save_local("index/faiss") # 5. Sparse BM25 index (keyword search fallback) tokenised = [c.page_content.lower().split() for c in chunks] bm25 = BM25Okapi(tokenised) with open("index/bm25.pkl", "wb") as f: pickle.dump({"bm25": bm25, "chunks": chunks}, f) return {"chunks": len(chunks), "status": "ok"}
Chunk size matters a lot. For academic papers, 512 tokens with 64 overlap captures one cohesive argument without splitting across method/result boundaries. Go smaller (256) for dense technical tables, larger (1024) for narrative review papers.
HyDE query expansion + hybrid retrieval + RRF
from langchain.vectorstores import FAISS from langchain.embeddings import OpenAIEmbeddings from sentence_transformers import CrossEncoder import pickle, openai def hyde_expand(cite_slot: str, paper_context: dict) -> str: """Ask LLM to write a hypothetical passage — embed this, not the raw query.""" prompt = f"""Write a 3-sentence passage that would appear in the {paper_context['section']} section of a {paper_context['type']} paper about {paper_context['topic']}, specifically addressing: {cite_slot}""" resp = openai.chat.completions.create( model="gpt-4o-mini", messages=[{"role":"user","content":prompt}] ) return resp.choices[0].message.content def hybrid_retrieve(cite_slot: str, ctx: dict, k: int = 20) -> list: # 1. HyDE expansion hypo_doc = hyde_expand(cite_slot, ctx) # 2. Dense retrieval (embedding similarity) embedder = OpenAIEmbeddings(model="text-embedding-3-small") vs = FAISS.load_local("index/faiss", embedder) dense_hits = vs.similarity_search_with_score(hypo_doc, k=k) # 3. Sparse retrieval (BM25 keyword) with open("index/bm25.pkl", "rb") as f: store = pickle.load(f) tokens = cite_slot.lower().split() bm25_scores = store["bm25"].get_scores(tokens) bm25_ranked = sorted(enumerate(bm25_scores), key=lambda x: -x[1])[:k] # 4. Reciprocal rank fusion K = 60 scores = {} for rank, (doc, _) in enumerate(dense_hits): cid = doc.metadata["chunk_id"] scores[cid] = scores.get(cid, 0) + 1 / (K + rank) for rank, (cid, _) in enumerate(bm25_ranked): scores[cid] = scores.get(cid, 0) + 1 / (K + rank) # 5. Cross-encoder rerank on top-20 candidates chunks = store["chunks"] top20 = sorted(scores, key=lambda c: -scores[c])[:20] reranker = CrossEncoder("cross-encoder/ms-marco-MiniLM-L-6-v2") pairs = [(cite_slot, chunks[c].page_content) for c in top20] rerank_scores = reranker.predict(pairs) ranked = sorted(zip(top20, rerank_scores), key=lambda x: -x[1]) return [chunks[c] for c, _ in ranked[:5]] # final top-5
The cross-encoder (ms-marco-MiniLM-L-6-v2) runs locally in ~30ms per pair on CPU. For a 20-candidate rerank, that's under 1 second — acceptable for batch section generation. Switch to a larger model (L-12) for higher-stakes reranking.
Tri-engine fusion and section writer
import asyncio from tavily import TavilyClient # web search async def web_search(query: str) -> list[str]: client = TavilyClient(api_key="...") results = client.search(query, max_results=5) return [r["content"] for r in results["results"]] async def generate_section(section_slot: dict, paper_ctx: dict) -> str: cite_slots = section_slot.get("cite_slots", []) # Run all three sources concurrently rag_tasks = [hybrid_retrieve(slot, paper_ctx) for slot in cite_slots] web_task = web_search(f"{paper_ctx['topic']} {section_slot['title']} recent") rag_results, web_results = await asyncio.gather( asyncio.gather(*rag_tasks), web_task ) # Flatten RAG chunks, deduplicate by chunk_id seen, rag_chunks = set(), [] for hits in rag_results: for doc in hits: cid = doc.metadata["chunk_id"] if cid not in seen: seen.add(cid); rag_chunks.append(doc.page_content) # Compose context block for writer LLM context = "\n\n".join([ "## Retrieved literature\n" + "\n---\n".join(rag_chunks[:8]), "## Web search findings\n" + "\n---\n".join(web_results), ]) writer_prompt = f"""You are writing the '{section_slot['title']}' section of a {paper_ctx['type']} paper on {paper_ctx['topic']}. Target length: {section_slot['word_budget']} words. Subsections: {section_slot['subheadings']} Citation slots to fill: {cite_slots} Use the retrieved context below as primary evidence. Add your own synthesis where context is insufficient. Insert [CITE:source] markers inline. Do NOT hallucinate statistics. {context} Write the section now:""" draft = openai.chat.completions.create( model="gpt-4o", messages=[{"role":"user","content":writer_prompt}] ).choices[0].message.content # Self-critique pass critique_prompt = f"""Review this draft section for: 1. Unsupported factual claims (flag with [UNSUPPORTED]) 2. Missing citations where evidence was available 3. Logical gaps between paragraphs 4. Deviation from {section_slot['word_budget']}-word target Draft:\n{draft}\n\nReturn the improved draft only.""" revised = openai.chat.completions.create( model="gpt-4o", messages=[{"role":"user","content":critique_prompt}] ).choices[0].message.content return revised
Run generate_section sequentially per section (Introduction → Related Work → Methods → Results → Discussion → Conclusion) and append to a manuscript dict. The sequential order matters — later sections can reference earlier ones in the context.
Vector store
FAISS recommended
Local, fast, free. Best for single-machine deployments with <1M chunks. No server needed — index lives on disk.
Chroma
Open-source, persistent, metadata-rich filtering. Good if you need SQL-style filtering alongside vector search.
Pinecone
Managed cloud service. Best for multi-user SaaS deployments or when you need serverless scaling.
Embedding model
text-embedding-3-small recommended
1536-dim, cheap ($0.02/M tokens), excellent quality. Best cost/quality tradeoff for academic paper RAG.
text-embedding-3-large
3072-dim. Meaningfully better on technical/scientific domains. Use if retrieval quality is a bottleneck.
BGE-M3 (local)
Open-source, multilingual, runs on GPU/CPU. Best for offline / data-private deployments on your own hardware.
Reranker
ms-marco-MiniLM-L-6 recommended
~30ms per pair on CPU. Excellent for the 20-candidate rerank step. Free, runs locally.
Cohere Rerank
API-based, higher accuracy, no local GPU needed. Best for cloud deployments with latency budget.
Web search
Tavily recommended
Research-optimised API, returns clean text (not HTML), citation-friendly. 1000 free searches/month.
Serper / SerpAPI
Google results. More coverage, but returns snippets only — need extra fetching for full content.
Orchestration
LangChain recommended
All loaders, splitters, retrievers, and chains already built. Fastest path to a working pipeline.
LlamaIndex
More RAG-specific abstractions (query engines, node parsers, knowledge graphs). Better for complex retrieval graphs.
Raw Python
Maximum control, no framework lock-in. Use if your retrieval logic is highly custom (e.g. KG-augmented RAG).
